# 支援プログラムLab マーケットプレイス設計書

## 目的

支援プログラムLabを、福祉事業所向け教材の閲覧サイトから、支援者・事業所・講師が自作プログラムを共有し、無料公開または有料販売できるプラットフォームへ発展させる。

初期の主な購入形態は次の2つとする。

- 単品購入: 1つの教材・プログラムを購入する
- セット購入: 就労支援セット、生活管理セット、コミュニケーションセットなど、複数教材をまとめて購入する

## 基本方針

現在のGitHub Pagesサイトは、外部向けサンプル・コンセプト確認用として維持する。本番機能であるログイン、出品、審査、決済、購入履歴、ダウンロード制御は、別の本番アプリとして構築する。

理由は、静的サイトに決済や個人情報を載せると、セキュリティ・運用・法務の境界が曖昧になるため。本番アプリでは、教材、出品者、購入者、ライセンス、注文、購入権限、審査履歴をデータベースで管理する。

## 推奨アプローチ

### 採用案: 審査制マーケットプレイス

出品者は教材をアップロードできるが、公開前に運営が審査する。購入者は、無料教材、単品教材、セット教材を探して利用できる。

この方式を採用する理由:

- 著作権侵害や個人情報入り教材の公開を防ぎやすい
- 福祉分野に必要な品質・表現・安全性を守りやすい
- 出品者を増やしても、初期のブランド毀損リスクを抑えられる
- 有料販売に移行しても、ライセンスと購入権限をシステムで管理できる

### 見送る案1: 完全オープン投稿型

誰でも即時公開できる方式。成長は速いが、著作権、個人情報、低品質教材、誤解を招く表現のリスクが高い。福祉教材では初期採用しない。

### 見送る案2: 運営だけが販売する教材ストア

法務・品質管理はしやすいが、現場の支援者が作成した教材を広く流通させるプラットフォーム性が弱い。初期の公式教材販売には使えるが、長期構想の中心にはしない。

## 法務・運用前提

この設計は法的助言ではない。Paid Beta開始前に、弁護士または専門家へ次の文書を確認してもらう。

- 利用規約
- 出品者規約
- 購入者ライセンス
- プライバシーポリシー
- 特定商取引法に基づく表示
- 返金・キャンセルポリシー
- 著作権侵害申し立てフロー
- 決済・売上分配フロー

初期実装で守る前提:

- 出品者は、自分が作成した教材、または販売許諾を持つ教材だけを出品できる
- 他者の本、画像、イラスト、動画、音楽、Web教材、研修資料の無断利用は禁止する
- 実在の利用者名、写真、支援記録、アセスメント、診断名、障害特性、事業所名など、個人を識別できる情報を含む教材は禁止する
- 医療、心理療法、診断、就職保証、報酬・加算取得保証などの効果を断定しない
- 購入者には、改変可否、クレジット表記、二次配布禁止、商用利用可否を購入前に明示する
- プラットフォーム内ポイント、預かり金、ウォレットはMVPでは扱わない

## ユーザー種別

### 購入者

福祉事業所、支援員、講師、管理者。教材を検索し、無料ダウンロード、単品購入、セット購入を行う。

主な操作:

- 教材を探す
- ライセンス条件を見る
- 無料教材をダウンロードする
- 単品教材を購入する
- セット教材を購入する
- 購入済み教材を再ダウンロードする

### 出品者

教材を作成した支援者、講師、事業所、法人。無料公開または有料販売のために教材を登録する。

主な操作:

- 出品者プロフィールを作成する
- 教材を下書き保存する
- ファイルをアップロードする
- ライセンスを選ぶ
- 単品価格を設定する
- 複数教材をセット化する
- 審査へ提出する
- 売上とダウンロード数を見る

### 運営管理者

審査、問い合わせ、違反対応、返金補助、カテゴリ管理を担当する。

主な操作:

- 提出教材を審査する
- 却下理由を記録する
- 公開停止する
- 著作権侵害申し立てを処理する
- 個人情報混入の疑いがある教材を隔離する
- カテゴリや特集を編集する

## 教材モデル

教材は、単体で購入・ダウンロードできる最小単位とする。

主な項目:

- タイトル
- 概要
- カテゴリ
- 対象サービス種別
- 所要時間
- 難易度
- 含まれるファイル
- 講師用資料の有無
- 利用者用ワークの有無
- 実施記録文例の有無
- 無料/有料
- 価格
- ライセンス
- 審査状態

審査状態:

- draft: 下書き
- submitted: 審査提出済み
- approved: 公開可能
- rejected: 差し戻し
- archived: 非公開

## セットモデル

セットは複数教材をまとめた商品とする。例は以下。

- 就労支援セット
- ビジネスマナー基礎セット
- 金銭管理・生活管理セット
- 新人職員向け講師台本セット
- B型事業所向け月間プログラムセット

MVPの制約:

- 1つのセットに含められる教材は、同じ出品者の承認済み教材のみ
- 複数出品者の教材を混ぜたセットはPost-MVP
- セット購入時は、含まれる全教材の購入権限を付与する
- 単品価格合計とセット価格を表示する
- セット限定教材はPost-MVPで扱う

## ライセンス設計

標準ライセンスは3種類。

### STANDARD_FREE

無料教材向け。無料閲覧・無料ダウンロードを許可するが、他法人への再配布・転売は禁止する。

### STANDARD_EDIT

福祉現場での利用を想定した標準ライセンス。購入した事業所内で、利用者に合わせたルビ、文字サイズ、説明文、順番の調整を許可する。クレジット表記は原則必須。再配布・転売は禁止。

### CUSTOM

特別な条件を設定するライセンス。運営審査を必須とする。初期は一般出品者には開放しない。

購入画面では必ず次を表示する。

- 改変可否
- クレジット表記の要否
- 編集用データの有無
- 他法人への二次配布・転売禁止
- 一般商用利用の可否

## 購入・権限設計

購入後は、注文そのものではなく、教材単位の購入権限を発行する。

例:

- 教材Aを単品購入した場合: 教材Aの購入権限を発行
- セットSに教材A/B/Cが含まれる場合: 教材A、教材B、教材Cそれぞれの購入権限を発行

この設計により、購入者のライブラリでは単品購入とセット購入の違いを意識せず、教材単位で再ダウンロードできる。

## 決済設計

MVPでは、1回の購入に含まれる出品者は1人に限定する。これにより、返金、売上分配、問い合わせ対応、特定商取引法表示を単純化する。

推奨:

- Stripe Connectなどのマーケットプレイス決済を利用する
- プラットフォームは手数料を受け取る
- 出品者への本人確認・売上支払いは決済サービス側の仕組みを使う
- 自前のウォレットや預かり金は作らない

将来:

- 複数出品者カート
- 法人請求書払い
- 月額サブスクリプション
- 事業所単位ライセンス

## 審査設計

公開前に次を確認する。

- 他者著作物の無断利用が疑われないか
- 個人情報・支援記録・写真が含まれていないか
- 差別的、威圧的、過度に矯正的な表現がないか
- 医療・診断・就職保証などを断定していないか
- ライセンスとファイル内容が矛盾していないか
- 価格と内容量が極端に不自然ではないか

審査結果:

- approve: 公開
- reject: 理由つき差し戻し
- suspend: 公開停止
- needs_legal_review: 法務確認待ち

## リリース段階

### Phase 0: 静的サンプル

現在のGitHub Pagesサイト。コンセプト、無料サンプル、ライセンス方針を見せる。

### Phase 1: クローズド無料版

ログイン、無料教材ライブラリ、運営作成教材のダウンロードを実装する。

### Phase 2: 審査制出品ベータ

限られた出品者だけが教材を登録できる。販売はまだ行わず、無料公開または限定公開で運用する。

### Phase 3: 有料単品販売

単品購入、購入権限、ダウンロード制御、返金処理を実装する。

### Phase 4: 有料セット販売

同一出品者内のセット販売を開始する。

### Phase 5: 公開マーケットプレイス

出品者募集を広げ、検索、レビュー、特集、売上分析を強化する。
