# 支援プログラムLab 無料共有プラットフォーム設計書

## 目的

支援プログラムLabを、福祉事業所向けの教材を無料で閲覧・ダウンロード・共有できるコミュニティ型プラットフォームとして育てる。

初期段階では、有料販売よりも人を集めることを優先する。投稿者が自作した教材を公開し、利用者がいいね、コメント、保存、ダウンロードを通じて反応できる状態を作る。

## 今回のMVP方針

- すべての教材は無料で閲覧・ダウンロードできる
- 投稿者は教材タイトル、概要、本文、イメージ画像、資料ファイル、対象サービス、所要時間、ライセンスを登録できる想定にする
- 投稿後は運営審査を通して公開する
- コメントといいねは、教材の使いやすさや現場での反応を集めるために見せる
- 画面上では資料ファイルと表紙画像を選択できるようにする
- 保存、ログイン、アップロード先ストレージ、コメント投稿の永続化は本格実装で接続する
- 決済、単品購入、セット購入、売上分配は後の実装として設計に残す

## 有料版の扱い

有料版は今回実装しない。ただし、将来のために次の設計は残す。

- 単品販売: 1つの教材を個別に購入できる
- セット販売: 就労支援セット、生活管理セット、コミュニケーションセットなど、複数教材をまとめて購入できる
- 将来価格案: 無料公開中の教材にも、後から有料化を検討するための価格メモを持たせる
- 購入権限: 単品購入でもセット購入でも、最終的には教材単位の利用権限を付与する
- 決済と売上分配: Stripe Connectなどのマーケットプレイス決済を使う想定で、自前ウォレットは作らない
- 法務確認: 有料化前に利用規約、出品者規約、特商法表示、返金ポリシー、著作権侵害対応を専門家へ確認する

有料化の判断は、無料版で投稿者数、ダウンロード数、コメント数、継続利用、問い合わせ内容を見てから行う。

## ユーザー像

### 利用者

福祉事業所の支援員、講師、サービス管理責任者、管理者。教材を探し、現場で使える資料を無料で保存・ダウンロードする。

主な行動:

- 教材一覧を読む
- カテゴリや人気順で探す
- 教材詳細を見る
- いいねする
- コメントを読む、将来はコメントする
- 無料でライブラリに追加する

### 投稿者

教材を作った支援者、講師、事業所、法人。自作教材を共有し、現場からの反応を得る。

主な行動:

- 教材タイトルと概要を書く
- 本文や実施方法を書く
- イメージ画像を添付する
- 講師用台本、利用者ワーク、PPT、Word、PDFなどを添付する
- 無料公開として審査に出す
- 将来の有料化メモを任意で残す

### 運営

投稿教材を審査し、公開品質と安全性を守る。

主な行動:

- 著作権侵害がないか確認する
- 個人情報や支援記録が含まれていないか確認する
- 利用者を責める、脅す、過度に矯正する表現がないか確認する
- 医療、診断、就職保証、加算取得保証に見える表現がないか確認する
- 公開、差し戻し、法務確認の判断をする

## 画面構成

### 教材一覧

教材投稿が読みやすく縦に並ぶフィードにする。

表示する内容:

- 投稿者
- 投稿日
- カテゴリ
- タイトル
- 概要
- ライセンスバッジ
- いいね数
- コメント数
- ダウンロード数
- 代表コメント
- イメージ画像またはカテゴリカード
- 将来価格案がある場合は未実装メモとして表示

### 教材詳細

教材の説明、対象サービス、含まれるファイル、補助資料、コメントを見せる。

無料ダウンロードの導線を主役にする。有料価格はあくまで将来メモとして扱う。

### 投稿フォーム

実装時に必要な項目が分かる投稿画面にする。

入力項目:

- 教材タイトル
- カテゴリ
- 概要
- 本文・使い方
- 対象サービス
- 公開設定
- 将来の有料版メモ
- イメージ画像
- 資料ファイル
- ライセンス
- 公開前チェック

### 投稿者ダッシュボード

投稿者が下書き、審査中、公開中、差し戻しを確認できる画面にする。

### 運営審査画面

提出された教材を、無料公開前に審査する画面にする。価格審査よりも、著作権、個人情報、表現安全性、ライセンスを優先する。

## データ方針

現在の静的プロトタイプでは、教材データをTypeScriptの配列で持つ。

教材データに含める主な項目:

- id
- slug
- title
- category
- sellerName
- summary
- description
- durationMinutes
- targetServices
- includedFiles
- supportItems
- priceYen
- futurePriceYen
- accessType
- authorAvatar
- coverTone
- likes
- downloads
- comments
- publishedAt
- license

`priceYen`は現在の公開価格を表す。無料MVPでは0円に統一する。`futurePriceYen`は将来の有料化検討用で、決済や購入には使わない。

## 実装範囲

今回作るもの:

- 無料教材フィードの強化
- 教材詳細の反応表示強化
- 投稿フォームの無料公開前提化
- 投稿者ダッシュボードの無料公開前提化
- 運営審査画面の無料公開前提化
- 無料版と有料版の境界を設計書に記録

今回作らないもの:

- ログイン
- 本物のファイルアップロード
- DB保存
- コメント投稿の保存
- いいねの保存
- 決済
- 売上管理
- 出品者本人確認
- 有料ダウンロード制御

## 成功指標

無料版で先に見る指標:

- 無料教材の閲覧数
- 無料ダウンロード数
- いいね数
- コメント数
- 投稿希望者数
- 審査通過教材数
- 事業所からの問い合わせ数

有料化判断で見る指標:

- 特定カテゴリの継続利用
- 同じ投稿者の教材への反応
- セット化してほしい要望
- 価格を払っても使いたいという声
- 運営審査にかかる負担

## リリース順

1. GitHub Pages上で無料共有プラットフォームの見た目を作る
2. 無料教材を増やし、投稿者募集の導線を見せる
3. 実際の投稿・アップロード・審査を本番アプリで実装する
4. 利用データと投稿者の反応を見て、有料単品販売を検討する
5. 単品販売が成立してから、セット販売を実装する

## 注意事項

この設計は法的助言ではない。有料販売を開始する前に、利用規約、ライセンス、特商法表示、返金、著作権侵害対応、個人情報保護の確認が必要。

無料版でも、個人情報や支援記録、他者著作物を含む教材は公開しない。福祉現場で使う教材として、利用者を責める表現や効果を保証する表現は避ける。
